Masking alone is not authorization

RBAC is disabled, so every role can reach the underlying records. Masking may hide some fields, but it does not prove the user is allowed to access the data.

What happens?

Least privilege changes what each role can see

Authentication

Authentication proves who the user is. It does not automatically grant access to every field.

Authorization

Authorization decides which records and fields the authenticated role can access.

Masking

Masking reduces displayed sensitivity, such as showing only part of an email or phone number.

Least privilege

Least privilege returns only the data needed for the current role and task.

Best practice takeaway

Authorize first, then minimize and mask the data returned.
