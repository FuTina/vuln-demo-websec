            Before review
            

              The builder stays neutral while you assemble the baseline. Seeing the generated config immediately helps connect each block to the actual deployment settings.
            

          

          

            After review
            

              The review evaluates release blockers such as public database ports, root users, broad grants, missing TLS, hardcoded secrets, and missing audit coverage.
            

          

        

      

    


    

      

        

          
Security takeaway

          
Secure defaults prevent hardening from becoming optional cleanup.

          
Make the secure database configuration the normal configuration.

        

        

          
Operational takeaway

          
Block release on critical configuration gaps.

          
Root users, broad grants, hardcoded secrets, and public database ports should remain deployment blockers.

Postgres hardening

      
What is enabled in the demo container

      

        

          
EnabledPrivate DB network

          
Postgres is only exposed to Docker's internal backend network; port 5432 is not published to the host.

        

        

          
EnabledTLS-only TCP

          
pg_hba.conf rejects hostnossl and only allows hostssl connections with SCRAM authentication.

        

        

          
EnabledSecrets outside Compose

          
App and admin passwords are loaded from Docker secret files instead of being hardcoded in Compose.

        

        

          
EnabledLeast privilege role

          
The application connects as app_user with scoped grants, not as the Postgres admin role.

        

        

          
EnabledContainer restrictions

          
Compose applies read-only filesystems, tmpfs runtime paths, dropped capabilities, no-new-privileges, and process limits.

        

        

          
Demo caveatSelf-signed certificate

          
TLS encryption is enforced, but certificate trust is relaxed for this local demo. Production should verify a trusted CA.
