What changed?

      
String concatenation lets input become SQL logic

      

        

          Vulnerable
          

            The login builds one SQL string with user-controlled text. Payloads such as alice' -- can
            comment out the password check, so the wrong password still grants access.
          

        

        

          Protected
          

            The login keeps SQL structure fixed with placeholders. The same payload is handled as a literal username,
            so it does not alter the query and the login fails.
          

        

      

    


    

      
Security control that fixes it

      
Use prepared statements for every database query

      

        Prepared statements separate SQL code from values. Validation and rate limits still matter, but they do not
        replace parameterized queries.
      

    


    

      
Best practice takeaway

      
Never concatenate login input into SQL.

      
No destructive payloads are needed to demonstrate the risk: read-only login bypass is enough.
