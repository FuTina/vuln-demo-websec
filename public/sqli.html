            Vulnerable
            

              The login builds one SQL string with user-controlled text. Payloads such as alice' -- can
              comment out the password check in vulnerable string-built SQL. The same class of bug can appear in any
              language when values are concatenated into SQL.
            

          

          

            Protected
            

              The login keeps SQL structure fixed with placeholders. The same payload is handled as a literal username,
              so it does not alter the query and the login fails.
            

          

        

      

      

        

          
Security takeaway

          
Prepared statements prevent user-controlled query manipulation.

        

        

          
Operational takeaway

          
Make parameterized queries a default code review requirement.
