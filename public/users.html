Users (rollenbasierte Ansicht)

Rolle: …

Du siehst aktuell die User‑Sicht (maskierte Daten). Für die Admin‑Sicht (volle Felder) starte den Server so neu:

macOS/Linux

DEMO_ROLE=admin npm start

Windows PowerShell

$env:DEMO_ROLE="admin"; npm start

Windows CMD

set DEMO_ROLE=admin && npm start
Was wird hier demonstriert?
  • User‑Rolle: E‑Mail & Telefon sind maskiert (Privacy by Default).
  • Admin‑Rolle: Volle Felder sichtbar (Need‑to‑Know).
  • Praxis: Zugriff auf sensible Daten wird über Rollen & Rechte gesteuert – nicht über Schwachstellen.
ID Name E‑Mail Telefon Adresse
