Cross-Site Scripting Rendering

Comments are harmless only if the application renders them safely. In vulnerable mode, demo links and a fake download become clickable page elements. In safe mode, the exact same input stays visible as text.

Encode output by default

innerHTML interprets markup

When untrusted input is assigned to innerHTML, the browser parses tags as page structure. Links and downloads become active UI.

textContent treats input as text

When assigned to textContent, angle brackets remain characters, not HTML.

Output encoding prevents injection

Encoding or safe rendering ensures user content is displayed in the intended context only.

Security takeaway

Default to safe text output for user-generated content.

If rich text is required, sanitize it with a proven library and pair it with CSP.
