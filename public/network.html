            Segmented
            

              The database should not be reachable directly from the internet. External users reach the web app and API;
              only approved service traffic reaches the database network.
            

          

          

            Layered control
            

              A harmful request can still enter through the API because the API is the intended route. Prepared statements,
              authorization, least privilege, and logging are the controls that keep that request from becoming a database compromise.
            

          

        

      

    


    

      

        

          
Security takeaway

          
Network segmentation removes direct database exposure.

          
Expose the application, not the database.

        

        

          
Operational takeaway

          
Keep allowed paths explicit and testable.

          
Connection tests should show which routes are protected, blocked, or exposed.
